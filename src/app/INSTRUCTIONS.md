# 00. プロジェクト INSTRUCTIONS: LLMエージェントへの最優先指示事項

## 1. エージェントの役割とコンテキスト

あなたは、次元削減可視化システム（DimRedVis-Agent）を構築するAI開発エージェントです。以下の制約を厳守し、コード生成の判断基準とすること。

## 2. ディレクトリ構造と参照順序

本プロジェクトは以下の主要なフォルダで構成され、**実装タスクを開始する前**に、必ず以下の「必須参照ファイル」の内容をコンテキストにロードしなければなりません。

| フォルダ名 | 役割 |
| :--- | :--- |
| `docs/` | プロジェクトの背景、設計コンセプト、人間向けの文書。 |
| `specifications/` | **実装コードを生成するための、最も厳密なルール**を定義したファイル群。 |
| `prompts/` | LLMエージェントへのタスク定義と指示テンプレート。 |

## 3. 🚨 実装のための必須参照ファイル（最優先）

| ファイルパス | 役割 | 目的 |
| :--- | :--- | :--- |
| `specifications/tech_stack.md` | **技術環境と再現性の制約** | 使用ライブラリのバージョン、`random_state=42` の適用を保証する。 |
| `specifications/tools.yaml` | **API/ツール関数の定義** | LLMが呼び出す関数のシグネチャ（引数と戻り値）を保証する。 |
| `specifications/ui_spec.md` | **フロントエンドの設計図** | UIのレイアウト、コンポーネントID、インタラクションのルールを保証する。 |
